This is the API

acceptProposal

Accept the proposal (possibly on behalf of the trust). Note that it is not neccessary to call this API in the case of direct-book scenarios as it will be assumed as a part of the workflow.

URL

http://healthsuite.allocatesoftware.com/api/v1/proposals/{id}/accept

HTTP Method

POST

Response Type

Parameters

  • Path: id String

    The ID of the proposal

cancelProposal

Cancel the proposal (and the booking, if it has been accepted)

URL

http://healthsuite.allocatesoftware.com/api/v1/proposals/{id}

HTTP Method

DELETE

Response Type

Parameters

  • Path: id String

    The ID of the proposal

getProposal

URL

http://healthsuite.allocatesoftware.com/api/v1/proposals/{id}

HTTP Method

GET

Response Type

ProposalDetails

Parameters

  • Path: id String

    The ID of the proposal

updateProposal

URL

http://healthsuite.allocatesoftware.com/api/v1/proposals/{id}

HTTP Method

PUT

Response Type

Parameters

  • Path: id String

    The ID of the proposal

  • Body: ProposalDetails ProposalDetails(ProposalDetails)

    Details of the proposal for filling the duty (worker)
