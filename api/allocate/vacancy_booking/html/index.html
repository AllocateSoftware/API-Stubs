Up 
post /proposals/{id}/accept
(acceptProposal)
Accept the proposal (possibly on behalf of the trust). Note that it is not neccessary to call this API in the case of direct-book scenarios as it will be assumed as a part of the workflow.

Path parameters

id (required)
Path Parameter — The ID of the proposal default: null

Responses

404

The proposal was not found

200

Proposal successfully accepted

AsyncResponse - Up

Where an API may respond with an asynchronous 'Accepted' (HTTP 202) response, it will return an AsyncResponse object. The eventual response will be returned by way of an async event, containing the corresponding correlation ID header returned below.

            The location of the correlation ID will depend upon the eventing transport in use - though typically it will be included as a message header (rather than a payload).
